01 · El Contexto

Los LLMs están aislados

Por sí solos, los modelos de lenguaje no pueden interactuar directamente con sistemas externos. No pueden consultar tu base de datos en tiempo real, no pueden ejecutar código fuera de su entorno, y no pueden navegar por internet.

🗄️

Bases de datos

Sin acceso a información en tiempo real.

⚡

Ejecución

Sin capacidad de correr código externo.

🌐

Internet

Sin poder navegar por la web por sí solos.

02 · Concepto Clave

¿Qué es una API?

Antes de ver la solución, necesitamos entender cómo se comunican los sistemas de software entre sí.

🍽️

La analogía del restaurante

No entras a la cocina a preparar tu comida. Le pides al mesero (la API) lo que quieres, él va a la cocina y te trae el resultado. Es el intermediario perfecto.

⛅

El ejemplo del clima

Una app del clima no tiene satélites propios. Llama a la API de un servicio meteorológico, pide los datos y te los muestra. Simple y poderoso.

"MCP convierte a la IA en un sistema capaz de actuar, no solo de responder."

03 · La Solución

El Model Context Protocol (MCP)

Teniendo claro el problema (LLMs aislados) y el concepto base (APIs como puentes), ahora sí podemos hablar de la solución definitiva.

🔌

Un protocolo abierto universal

MCP permite conectar modelos de inteligencia artificial con herramientas externas de forma estandarizada: APIs, bases de datos, navegadores, sistemas de archivos, y mucho más.

Y la frase que mejor lo resume es esta:

"MCP convierte a la IA en un sistema capaz de actuar, no solo de responder."

04 · Estructura

Arquitectura Cliente-Servidor

MCP funciona sobre un modelo muy común en el desarrollo de software. Hay tres actores principales que hacen posible la magia.

"MCP actúa como el puente entre la inteligencia artificial y el mundo real."
🤖

1. El Cliente

En nuestro caso es Claude Code — la IA con la que interactúas directamente.

🌉

2. El Servidor MCP

Actúa como el intermediario inteligente que traduce las necesidades de la IA.

🛠️

3. Las Herramientas

Los servicios externos: APIs, bases de datos, navegadores, lo que necesites.

05 · Funcionamiento

¿Cómo fluye la información?

El proceso funciona así, y es más simple de lo que parece.

1. Solicitud

Petición en lenguaje natural.

2. Detección

El modelo identifica la herramienta necesaria.

3. Envío

MCP enruta la solicitud al servidor.

4. Ejecución

El servidor realiza la acción.

5. Respuesta

El modelo recibe los datos y responde.

06 · Capacidades

¿Qué puede ofrecer un servidor MCP?

Un servidor MCP dota al modelo de tres superpoderes fundamentales:

🛠️

1. Acceso a Herramientas

Conexión a APIs externas, servicios de pago, redes sociales y cualquier sistema interactivo.

⚡

2. Ejecución de Acciones

Capacidad para correr código, automatizar procesos complejos y controlar navegadores web.

🧠

3. Provisión de Contexto

Inyección de datos dinámicos y actualizados (estado de una BD o contenido web en tiempo real).

07 · Herramientas

Algunos tipos de MCP y sus funciones

🎭

Playwright MCP

Automatización de navegadores

Controla un navegador real. Abre páginas, hace scroll, llena formularios y toma screenshots. Todo ejecutado de verdad, sin simulaciones.

🎨

Figma MCP

De diseño a código

Accede a tus diseños y los convierte en código funcional. Extrae variables y componentes respetando tu sistema de diseño, e incluso escribe en el canvas.

📚

Context7 MCP

Documentación actualizada

Inyecta documentación oficial en tiempo real (Next.js, React, etc.) al modelo, evitando errores por APIs obsoletas.

El Poder de Combinar

Y lo más interesante de todo es que puedes combinarlos. Claude puede en una sola instrucción consultar tu base de datos, verificar algo en una página web, generar código que respete tu sistema de diseño de Figma, y hacerlo todo con documentación actualizada gracias a Context7.

Todo coordinado. Todo automatizado.

08 · Demo en vivo

Manos a la obra

Vamos a conectar Claude Code con Playwright para automatizar un navegador en tiempo real.

01. Abrir Claude Code

Sin herramientas externas. El modelo no puede tocar el navegador todavía.

02. Instalar Playwright MCP

Un solo comando: npx @playwright/mcp@latest

03. Configurar mcp.json

Agregar el servidor en el archivo de configuración.

04. ¡Probar en vivo!

Le pedimos a Claude que abra Google y tome un screenshot. Sin código.

// mcp.json { "mcpServers": { "playwright": { "command": "npx", "args": ["-y", "@playwright/mcp@latest"] } } }
